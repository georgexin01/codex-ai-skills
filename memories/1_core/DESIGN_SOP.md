# 📐 wRider Design SOP (Standard Operating Procedure)
> 🤖 **MODEL SYNC**: This is the structural manifest for page-level architecture. Any AI model joining the session MUST replicate these blueprints for all new view creations.

> Detailed structural guides for building consistent, high-density industrial pages within the wRider ecosystem.

## 1. The "Transactional" Blueprint (Wallet / Payroll)
Used for financial displays and ledger-heavy views.

- **Hero Block**:
    - **Design**: Absolute container with deep gradient (`violet-dark`).
    - **Typo**: `text-4xl font-black tracking-tight` for the main amount.
    - **DNA**: Include a background watermark icon (Banknote/Wallet) at 160px size with 20% opacity.
- **KPI Strip**:
    - **Layout**: 3-column grid (`grid-cols-3 gap-2`).
    - **Design**: Solid `WRCard` (Tier 1 Paper).
    - **Typo**: `wr-eyebrow` title + `text-base font-bold` value.
- **List Structure**:
    - **Row Design**: `WRCard` with 40px icon box.
    - **Iconography**: `bg-teal-soft text-teal` (Income) vs `bg-pink/10 text-pink` (Expense/Withdrawal).
    - **Meta**: Use 10px uppercase tracking for Transaction IDs and Dates.

## 2. The "Operations" Blueprint (Home / Dashboard)
Used for live mission tracking and fleet monitoring.

- **Status Bar**: Always include the status spacer at the top.
- **Mission Hub**:
    - **Layout**: Full-width Balance Card with 4-action grid.
    - **DNA**: Use `Plus`, `Users`, `Camera`, `Banknote` as the primary operational glyphs.
- **The "Focus" Constraint**:
    - **Rule**: Only show **one** active mission hero.
    - **UX**: Use a "View more" link for secondary mission lists.
- **Progress Visualization**:
    - **Rule**: Bars must be **6px** height.
    - **Gradients**:
        - On-track: `sky-300 → sky-400`
        - Tight: `amber-200 → gold`
        - Late: `red-pink (#ff4f4f → #ef0063)`

## 3. The "Glass" Utility (Toasts / Modals / Overlays)
Used for transient feedback or focused entry (Login).

- **Backdrop**: `backdrop-filter: blur(12px)`.
- **Surface**: `rgba(255, 255, 255, 0.85)` (Admin) or slightly more opaque for Rider.
- **Border**: `1px solid rgba(255, 255, 255, 0.4)`.
- **Typography**: Force all child text to use **700-weight** for titles to maintain legibility against blurred backgrounds.

## 4. Hierarchy Mandates (The DNA Checklist)
1. **Header**: Every sub-page MUST have `WRHeader` with `back` prop enabled.
2. **Empty States**: Never show a blank screen; always use `WREmpty` with a contextual Lucide icon.
3. **Card Interaction**: Use `:clickable="true"` on `WRCard` for all list items. Remove standalone "View/Edit" text buttons.
4. **Spacing**: Top-level containers use `px-4 pt-4`; internal sections use `space-y-4`.
