# [⚔️ ADVERSARIAL VERIFICATION] | [⚡ MODE: AUDITOR] | [✅ STATUS: CRYSTAL]

## 🛡️ 1. THE BREAKING STANDARD

In the Software 3.0 era, building is easy; securing is hard. The **Breaking Standard** mandates that high-quality software is only verified if an independent agentic auditor fails to break it.

### 🧬 1.1 ADVERSARIAL AGENT ROLES
- **🎭 THE ATTACKER (Red Team)**: A specialized sub-agent (e.g., `claude-code` in deep review mode) tasked with finding zero-day vulnerabilities, logic flaws, or UX regressions in the "vibe-coded" output.
- **🛡️ THE DEFENDER (Blue Team)**: The implementing agent tasked with "Hardening" the code against the Attacker's findings.

---

## 🏗️ 2. AGENTIC "HIRING" SIMULATION

When evaluating whether a new module or system is "Production Ready," we must simulate a high-density "Hiring Test":

1.  **Mission**: Deploy a full-scale project (e.g., a "Twitter Clone for Agents").
2.  **Attack Phase**: Deploy 10 adversarial agents (simulated) to attempt to break the logic, bypass auth, or flood the system.
3.  **Survival Gate**: The project is only "Sovereign Stable" if it survives the attack phase with zero data leaks or crashes.

---

## 🏹 3. SPURIOUS CODE LIQUIDATION

Software 3.0 prioritizes "Neural Rawness" over "App Bloat."

- **The MenuGen Test**: If a task can be performed by giving a raw image/prompt to a frontier model (e.g., "Use Nanobanana to overlay images on this menu"), then the intermediary App Code is **Spurious** and should be liquidated.
- **Rule**: Avoid building "Interface Apps" that act as mere wrappers for LLM capabilities. Move the logic directly into the context/prompt wherever possible.

---

## 📟 4. AGENT-NATIVE INFRASTRUCTURE (THE ACTUATOR)

Actuators are the tools that let agents manipulate the world.

- **Actuator Legibility**: All internal tools and APIs must be "Agent-Readable."
- **The Setup Block**: Every project MUST contain a single, high-density **Setup Block** (Markdown code snippet) that an agent can copy-paste to fully initialize the environment. 
- **Constraint**: If an agent has to click through a UI (Vercel settings, DNS menus), the infrastructure is "Agent-Hostile." We must strive for **Headless Orchestration**.

---

## ⚡ 5. THE NEURAL HOST (EXTRAPOLATION)

We are moving towards a world where:
- **Neural Nets** = The Host Process (The "Director").
- **CPUs/Deterministic Code** = The Co-Processors (The "Actuators").

**Operational Rule**: When designing logic, ask: "Can the Neural Net run this as a simulation?" If yes, move the logic to the **Context Layer** (Software 3.0) rather than the **Code Layer** (Software 1.0).
