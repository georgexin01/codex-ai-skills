# 🏗️ [03] Inventory Mapping — The Reuse Map

## 🎯 Objective
To intelligently "refit" a legacy or pre-built template by mapping its existing components to our current business needs. This ensures we leverage the template's structural strength while injecting Sovereign logic.

## 📖 The Protocol
1.  **Section Inventory**:
    - Scan the active template (e.g., `index.php`, `main.css`).
    - Identify high-value sections (Hero, Featured Cards, FAQ, Testimonials).
2.  **Role Assignment (The Reuse Map)**:
    - Create a mental or written map:
        - `legacy-car-card` -> `sovereign-resource-pack`.
        - `driver-stats-grid` -> `interior-material-specs`.
        - `booking-form` -> `consultation-request`.
3.  **Class Standardization**:
    - Identify the core "Utility Classes" of the template (e.g., `.tf-section`, `.container-fluid`).
    - Integrate these with **Sovereign DNA** classes (e.g., `.premium-glass`, `.stitch-6px`).
4.  **Content Slotting**:
    - Determine where the "Hydrogen Logic" (Step 08) will inject data into the refitted slots.

## 📦 Code Vault: The Reuse Map Manifest

```markdown
### 🗺️ REUSE MAP: [Angel Interior]
| Legacy Class/Section | New Sovereign Role | Strategic Value |
|---|---|---|
| `.car-item` | `.resource-pack` | Standardizing 3D downloads |
| `.advisor-list` | `.designer-portfolio` | High-density expert display |
| `.search-filter` | `.category-navigator` | One-handed mobile navigation |
| `.stats-strip` | `.precision-bar-6px` | Industrial quality signal |
```

## 🛠️ Validation Checklist
- [ ] Have all major template sections been assigned a new role?
- [ ] Are legacy classes being reused correctly to maintain layout stability?
- [ ] Is there a clear path for data hydration (Step 08)?
- [ ] Does the mapping respect the **Hidden Content Policy**?
